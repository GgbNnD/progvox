# 开发上下文压缩记录

## 项目目标

实现一个生成式极低码率视频通话原型系统。主线采用 ProGVC 思路：发送端把帧转成多尺度残差 token maps，接收端按网络情况接收部分 token，缺失层由上下文模型和生成器补全，最终实现渐进式传输和低码率重建。

## 已实现模块

- `models/tokenizer.py`
  - `MultiScaleResidualTokenizer`
  - 4 层 token maps，默认 4/8/16/32 网格
  - 支持截断重建、码率估算和 zero-residual token id 查询

- `models/generator.py`
  - `DetailSynthesisCNN`
  - `PatchDiscriminator`
  - `TinyConditionalDiffusion`
  - 短程消融中 `cnn_gan` 当前最好

- `models/context_model.py`
  - `TokenContextTransformer`
  - `flatten_token_maps` / `unflatten_token_maps`
  - 已有正式训练脚本 `train/train_context_model.py`
  - 第一轮训练 checkpoint 本地保存在 `checkpoints/context_model/context_model.pth`

- `run_progvc_codec.py`
  - 离线模拟 tokenizer -> context fallback -> CNN-GAN generator
  - Layer 2 约 90.6 kbps；Layer 3 约 366.8 kbps

- `transport/scheduler.py`
  - `GreedyScheduler`
  - `SlidingWindowScheduler`
  - deadline-based trace 仿真
  - 默认 token 层 bits/frame：144、576、2304、9216

- `scripts/simulate_scheduler.py`
  - 生成固定弱网 trace 对比报告
  - 输出 `reports/scheduler_comparison.csv`
  - 输出 `reports/scheduler_comparison.json`
  - 输出 `reports/scheduler_trace.png`

- `models/abr_controller.py`
  - `RuleBasedABRController`
  - `NetworkObservation`
  - `ABRDecision`
  - `observations_from_trace`
  - 使用 throughput EWMA、RTT、loss rate、queue delay 和稳定升档滞后选择最大 token 层级

- `scripts/simulate_abr.py`
  - 生成 ABR 决策 CSV/JSON/PNG/Notebook
  - 将 ABR max-layer cap 接入 `SlidingWindowScheduler`

- `transport/webrtc_loopback.py`
  - `LoopbackConfig`
  - `SyntheticVideoTrack`
  - `run_loopback`
  - 同进程 aiortc offer/answer、DataChannel echo、合成视频轨道收帧

- `transport/datachannel_proto.py`
  - `TokenPacket`
  - `pack_packet` / `unpack_packet`
  - `fragment_payload`
  - `FrameReassembler`
  - 支持乱序重组、重复 chunk 统计、deadline/timeout 丢弃和 CRC32 校验

- `scripts/simulate_datachannel_proto.py`
  - 仿真 1%/3%/5% 丢包与乱序到达
  - 输出 `reports/datachannel_protocol_robustness.csv`
  - 输出 `reports/datachannel_protocol_robustness.json`

- `transport/offline_loopback.py`
  - `run_offline_loopback`
  - `token_map_to_payload` / `payload_to_token_map`
  - 加载 tokenizer、context checkpoint、CNN-GAN generator
  - 通过 ABR + scheduler simulation 决定每帧传输层级
  - 使用 DataChannel protocol 分片/重组 token payload
  - 输出码率、stall、PSNR/SSIM、延迟分解、显存和小样图

- `sender_main.py` / `receiver_main.py`
  - 当前作为一进程 offline loopback 原型入口
  - 后续可拆分为 live DataChannel sender/receiver

## 最新实验与测试状态

- 上下文模型训练：80 step，最终验证 loss 4.2248，perplexity 68.360，next-token accuracy 0.0410。
- 阶段 3.1 调度仿真：同一弱网 trace 下，贪婪调度 stall rate 10.0%，滑动窗口 3.3%；PSNR fluctuation 0.71 -> 0.47。
- 阶段 3.2 ABR 仿真：ABR-capped sliding window 发送码率 189.5 kbps，对比 uncapped sliding window 248.0 kbps；stall rate 均为 3.3%，render-PSNR proxy 20.86 -> 19.56。
- 阶段 4.1 WebRTC loopback：5/5 DataChannel ack，12 个视频帧收到，双方连接状态 connected。
- 阶段 4.2 protocol 仿真：1%/3%/5% 丢包下 decodable frame rate 分别为 99.2%、96.7%、92.5%；full frame rate 分别为 95.0%、82.5%、65.0%。
- 阶段 4.3 offline loopback：64 帧 foreman，发送码率 251.0 kbps，stall 6.25%，PSNR-RGB 17.58，SSIM-RGB 0.5175，估计端到端延迟 136.2 ms，峰值显存 284.1 MB。
- `pytest tests -q`：31 个测试通过，3 个 PyTorch nested tensor warning。
- 权重和 TensorBoard 日志在 `checkpoints/`、`runs/`，被 git 忽略。
- 当前已提交到阶段 4.2 节点：`9005322 feat(protocol): add datachannel token packets`
