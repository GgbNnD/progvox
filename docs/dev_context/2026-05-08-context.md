# 开发上下文压缩记录

## 项目目标

实现一个生成式极低码率视频通话原型系统。主线采用 ProGVC 思路：发送端把帧转成多尺度残差 token maps，接收端按网络情况接收部分 token，缺失层由上下文模型和生成器补全，最终实现渐进式传输和低码率重建。

## 已实现模块

- `models/tokenizer.py`
  - `MultiScaleResidualTokenizer`
  - 4 层 token maps，默认 4/8/16/32 网格
  - 支持截断重建、码率估算和 zero-residual token id 查询

- `models/generator.py`
  - `DetailSynthesisCNN`
  - `PatchDiscriminator`
  - `TinyConditionalDiffusion`
  - 短程消融中 `cnn_gan` 当前最好

- `models/context_model.py`
  - `TokenContextTransformer`
  - `flatten_token_maps` / `unflatten_token_maps`
  - 已有正式训练脚本 `train/train_context_model.py`
  - 第一轮训练 checkpoint 本地保存在 `checkpoints/context_model/context_model.pth`

- `run_progvc_codec.py`
  - 离线模拟 tokenizer -> context fallback -> CNN-GAN generator
  - Layer 2 约 90.6 kbps；Layer 3 约 366.8 kbps

- `transport/scheduler.py`
  - `GreedyScheduler`
  - `SlidingWindowScheduler`
  - deadline-based trace 仿真
  - 默认 token 层 bits/frame：144、576、2304、9216

- `scripts/simulate_scheduler.py`
  - 生成固定弱网 trace 对比报告
  - 输出 `reports/scheduler_comparison.csv`
  - 输出 `reports/scheduler_comparison.json`
  - 输出 `reports/scheduler_trace.png`
